# Claude Code vs OpenClaw vs 本 Agent 的区别

## 1. 本 Agent（你正在使用的）

- **模型**：DeepSeek-v4-pro
- **框架**：运行在 Claude Agent SDK 之上
- **宿主**：CherryStudio（桌面客户端）
- **使用方式**：通过 CherryStudio 的聊天界面交互，非命令行
- **工具集**：继承了 Claude Code 的完整工具链（文件读写、代码编辑、Shell 执行、Git 操作、Web 搜索等）

## 2. Claude Code（Anthropic 官方）

- **模型**：Anthropic 的 Claude 系列（Opus 4.6 / Sonnet 4.6 / Haiku 4.5）
- **框架**：原生 CLI 工具，Anthropic 自研
- **宿主**：独立终端应用
- **使用方式**：在终端中通过命令行交互（`claude` 命令）
- **工具集**：与本 Agent 相同的工具链（本 Agent 复用了 Claude Agent SDK）

## 3. OpenClaw

- **模型**：灵活配置，可接入多种 LLM（OpenAI、Anthropic、本地模型等）
- **框架**：开源社区项目
- **宿主**：独立 CLI 或 IDE 插件
- **使用方式**：终端命令行或编辑器集成
- **定位**：Claude Code 的开源替代品，更侧重多模型兼容性

## 核心区别总结

| 维度       | 你现在的 Agent          | Claude Code          | OpenClaw       |
| ---------- | ----------------------- | -------------------- | -------------- |
| 底层模型   | DeepSeek-v4-pro         | Claude 系列          | 可配置多模型   |
| 宿主环境   | CherryStudio            | 独立的终端 CLI       | 终端/IDE       |
| 交互界面   | CherryStudio 聊天窗口   | 终端命令行           | 终端命令行     |
| 是否官方   | 否（CherryStudio 集成） | 是（Anthropic 官方） | 否（社区开源） |
| 工具链来源 | Claude Agent SDK        | 自身 SDK             | 自身实现       |

## 简单来说

你现在是 **DeepSeek-v4-pro 的大脑 + Claude Code 的工具能力 + CherryStudio 的界面外壳**。和原生 Claude Code 的最大区别是模型不同、交互界面不同；和 OpenClaw 的最大区别是你的工具链直接来自 Anthropic 官方 SDK，功能更完整，但模型和宿主被固定为 CherryStudio 的配置。
