# 第 4 章 课堂练习 —— 实验四：磁盘和文件系统管理

## 实验目的

1. 掌握磁盘的添加、分区、格式化、挂载
2. 掌握磁盘阵列的管理
3. 掌握 LVM 的管理

## 实验内容

### （一）基础知识

**Linux 磁盘及分区的表示方法**

**Linux 文件系统：**

```bash
cat /etc/filesystems
```

**swap**

**查看分区信息：**

```bash
fdisk -l
```

给虚拟机添加 5 块新硬盘，重启后才能识别新硬盘。

**给新硬盘分区：**

```bash
fdisk /dev/sdb
```

- `n: p`，主分区，5G
- `n: e`，扩展分区（余下的所有空间）
- `n: l`，两个逻辑分区：sdb5 8G，sdb6
- `w`，写入分区信息并退出

**查看系统内核是否已经识别出新分区：**

```bash
cat /proc/partitions
```

**格式化分区**

```bash
mkfs -t xfs /dev/sdb1
mkfs -t ext4 /dev/sdb5
mkfs.xfs -f /dev/sdb6
```

**显示所有可用的块设备的文件系统：**

```bash
lsblk -f
```

**挂载**

查看挂载设备：`df -h`，看有无 sdb。

```bash
mkdir /data
mount /dev/sdb1 /data

mkdir /mnt/{game,movie}
mount /dev/sdb5 /mnt/game
mount /dev/sdb6 /mnt/movie
```

4）查看挂载：

```bash
df -hT | grep -v "tmpfs"
```

5）上述的挂载在重启时会自动被卸载，可修改 `/etc/fstab` 文件让系统自动挂载，如：

```
/dev/sdb1   /data      xfs   defaults  0  0
/dev/sdb5   /mnt/game  ext4  defaults  0  0
/dev/sdb6   /mnt/movie xfs   defaults  0  0
```

6）让 `/etc/fstab` 文件中的挂载立即生效：

```bash
mount -a
```

7）重启，用 `lsblk -f` 查看可用的块设备。

### 磁盘配额

**启用磁盘配额：** 以 `/dev/sdb1` 的挂载点 `/data` 为例。

编辑 `/etc/fstab`：

```
/dev/sdb1  /data  xfs  defaults,uquota,gquota  0  0
```

检查配额是否生效：

```bash
mount | grep sdb1   # 磁盘配额生效了没有？
```

启用磁盘配额后，要重新挂载才会生效：

```bash
umount /data    # 先卸载
mount -a       # 再挂载
mount | grep sdb1   # 查看
```

**设置用户配额**

设置用户 hd 的磁盘容量硬限制额为 100MB，文件数量硬限制额为 3 个：

```bash
[root@localhost ~]# edquota -u hd
```

设置用户组 hd 的磁盘容量硬限制额为 200MB，文件数量硬限制额为 6 个（只对基本组生效）：

```bash
[root@localhost ~]# edquota -g hd
```

添加新用户 jerry，指定其基本组为 hd：

```bash
[root@localhost ~]# useradd -g hd jerry
```

生成一个 60MB 的测试文件：

```bash
dd if=/dev/zero of=/tmp/test1 bs=1M count=60
```

生成一个 10MB 的测试文件：

```bash
dd if=/dev/zero of=/tmp/test2 bs=1M count=10
```

设置 `/data` 权限：

```bash
chmod 777 /data
```

**切换到 hd 验证：**

```bash
cp /tmp/test1 /data/a
cp /tmp/test1 /data/b       （这里是否报错？为什么？）
ll -h /data   # 查看/data目录下a和b两个是否存在？大小是多少？
```

此时，hd 的磁盘配额已用完，要删除 b 文件后，再做后面的测试：

```bash
rm /data/b
cp /tmp/test2 /data/b
cp /tmp/test2 /data/c
cp /tmp/test2 /data/d       （这里是否报错？为什么？）
```

**再用 jerry 验证：**

```bash
cp /tmp/test1 /data/a1
cp /tmp/test1 /data/a2  （这里是否报错？为什么？）
cp /tmp/test2 /data/a2
cp /tmp/test2 /data/a3
cp /tmp/test2 /data/a4 （这里是否报错？为什么？）
```

10）查看用户或分区的配额使用情况：

```bash
[root@localhost ~]# quota -u hd
[root@localhost ~]# quota -g hd
[root@localhost ~]# repquota /data
```

### （二）配置 RAID10

1. 为虚拟机添加 4 个硬盘，添加成功后，重启虚拟机。

2. 查看 4 个新硬盘的名称如：sdb sdc sdd sde

```bash
fdisk -l
```

3. 创建 RAID：

> 注意：如果接着前面已经挂载的 /dev/sdb 来做，需要将已经挂载的分区全部删除（用 `fdisk /dev/sdb` p－显示分区 d－删除分区 w-保存退出），/etc/fstab 文件中增加的条目也要删除。

```bash
mdadm -C /dev/md0 -a yes -n 4 -l 10 /dev/sd{b..e}
```

4. 查看磁盘阵列信息

```bash
cat /proc/mdstat
mdadm -D /dev/md0
```

5. 将磁盘阵列格式化为 XFS 文件系统

```bash
mkfs -t xfs /dev/md0
```

6. 创建挂载点后挂载磁盘阵列，并查看已挂载的设备

```bash
mkdir /raid
mount /dev/md0 /raid
df -hT | grep -v tmpfs
```

7. 从 `/dev/zero` 复制数据到 `/tmp/test1`，大小为 500 个 1MB 的数据块，查看对普通磁盘写入 500MB 数据，平均速度是多少

```bash
dd if=/dev/zero of=/tmp/test1 bs=1M count=500
```

8. 性能测试：对磁盘阵列进行同样的写入操作，看速度是多少

```bash
dd if=/dev/zero of=/raid/test1 bs=1M count=500
```

### （三）RAID 故障模拟

1. 向 `/raid` 中复制一个文件，并测试可以正常打开

```bash
cp /etc/passwd /raid
cat /raid/passwd
```

2. 将阵列中的某块硬盘标记为损坏

```bash
mdadm /dev/md0 -f /dev/sdc
```

3. 查看磁盘阵列信息，看是否提示有一块硬盘损坏

```bash
mdadm -D /dev/md0
```

4. 查看 `/raid` 中的 passwd，看是否受磁盘损坏的影响

```bash
echo "raid10" >> /raid/passwd
tail -1 /raid/passwd
```

5. 将损坏的硬盘从阵列中移除

```bash
mdadm /dev/md0 -r /dev/sdb
```

此时可再查看一下磁盘阵列的详细信息，以及原有文件是否受影响。

6. 向阵列添加一块新硬盘，查看磁盘阵列详细信息

```bash
mdadm /dev/md0 -a /dev/sdf
mdadm -D /dev/md0   （需要有一定时间重建数据）
```

7. 再查看原文件是否受影响

### （四）RAID5 和备份盘

1. 先将 `/raid` 挂载点卸载

```bash
umount /raid
```

2. 再停用前面创建的磁盘阵列

```bash
mdadm -S /dev/md0
```

3. 重新创建一个 RAID5 磁盘阵列

```bash
mdadm -C /dev/md1 -a yes -n 3 -l 5 -x 1 /dev/sd{b..e}
```

4. 查看磁盘阵列信息

```bash
mdadm -D /dev/md1     （需要等待一段时间才能完成重建数据）
```

5. 重新格式化

```bash
mkfs -t xfs -f /dev/md1
```

6. 重新挂载

```bash
mount /dev/md1 /raid
cp /etc/passwd /raid/passwd.bak  （复制一个文件作测试）
```

7. 把硬盘 sdc 标记为损坏，查看阵列信息

```bash
mdadm /dev/md1 -f /dev/sdc
```

```
mdadm: set /dev/sdc faulty in /dev/md1
```

```bash
mdadm -D /dev/md1  （此时，备份盘已自动替换上去，并正在重建数据）
```

8. 将坏掉的 sdc 移出磁盘阵列

```bash
mdadm /dev/md1 -r /dev/sdb
```

9. 重新为虚拟机添加一块新硬盘，然后重启，识别到为 sdf，将 sdf 重新格式化，再加入到 RAID5 阵列

```bash
mkfs -t xfs /dev/sdf
mdadm --add /dev/md1 /dev/sdf
```

10. 查看原来的文件是否还在

### （六）创建物理卷 LVM

如果（二）~（五）没有做，也先卸载并清除 sdb 上的现有挂载：

```bash
umount /dev/sdb{1,5,6}     # 卸载
```

再编辑 `/etc/fstab`，注释掉这几行：

```
#/dev/sdb1 /data xfs defaults,uquota,gquota 0 0
#/dev/sdb5 /mnt/game ext4 defaults 0 0
#/dev/sdb6 /mnt/movie xfs defaults 0 0
```

再擦除 sdb 的分区表：

```bash
dd if=/dev/zero of=/dev/sdb bs=512 count=1
```

> 注意：dd 命令非常危险！`of=/dev/sdb` 必须写对硬盘！写错成 sda 会直接干掉系统盘！

然后从 4.创建物理卷开始做。

1. 如果是接着前面的实验做，先停止 RAID

```bash
umount /raid
mdadm -S /dev/md1
```

2. 创建物理卷

```bash
[root@localhost ~]# pvcreate /dev/sd{b,c}
```

```
  Device /dev/sdb excluded by a filter.     // 如果有这样的报错信息，是因为前面创建RAID时，这些磁盘上还留有信息，此时，需要先将这些信息删除
  Device /dev/sdc excluded by a filter.
```

3. 删除磁盘原有信息

```bash
mdadm --zero-superblock /dev/sd{b,c}
```

4. 再创建物理卷 PV

```bash
pvcreate /dev/sd{b,c}
```

```
  Physical volume "/dev/sdb" successfully created.
  Physical volume "/dev/sdc" successfully created.
```

```bash
pvdisplay /dev/sd{b,c}    // 查看PV信息
```

5. 创建卷组 VG - wgroup

```bash
vgcreate wgroup /dev/sd{b,c}
```

```
  Volume group "wgroup" successfully created
```

```bash
vgdisplay wgroup   // 查看wgroup信息
```

6. 创建逻辑卷 LV - ftp

```bash
lvcreate -L 39G -n ftp wgroup
```

```
  Logical volume "ftp" created.
```

```bash
lvdisplay /dev/wgroup/ftp    // 查看ftp信息
```

7. 格式化逻辑卷 ftp

```bash
mkfs -t xfs /dev/wgroup/ftp
```

8. 创建挂载点目录，将逻辑卷挂载

```bash
mkdir /var/ftp
mount /dev/wgroup/ftp /var/ftp
df -hT | grep -v tmpfs      // 查看已挂载的分区信息
```

```
文件系统                类型      容量  已用  可用 已用% 挂载点
/dev/mapper/centos-root xfs        17G  5.5G   12G   32% /
/dev/sda1               xfs      1014M  172M  843M   17% /boot
/dev/sr0                iso9660   4.4G  4.4G     0  100% /run/media/root/CentOS 7 x86_64
/dev/mapper/wgroup-ftp  xfs        39G   33M   39G    1% /var/ftp
```

重启系统，这个挂载点消失，需要修改 `/etc/fstab`，实现自动（永久）挂载。

9. 使用

```bash
cd /var/ftp
ll
```

```
总用量 0
```

```bash
echo "test" >> test.txt
cat test.txt
```

10. 扩展逻辑卷空间

1）先将另两块硬盘组成物理卷 PV

```bash
pvcreate /dev/sd{d,e}        // 如果有报错，看2.和3.
```

2）再将此物理卷加入 wgroup

```bash
vgextend wgroup /dev/sd{d,e}
```

```
  Volume group "wgroup" successfully extended
```

3）再将逻辑卷 ftp 扩容（增加 10G）

```bash
lvextend -L +10G /dev/wgroup/ftp
```

```
  Size of logical volume wgroup/ftp changed from 39.00 GiB (9984 extents) to 49.00 GiB (12544 extents).
  Logical volume wgroup/ftp successfully resized.
```

4）再更新 ftp 的文件系统（不同的文件系统使用的命令不同，这里是 xfs）

```bash
xfs_growfs /dev/wgroup/ftp
```

5）再查看 ftp 信息，注意容量已经变成 49G

```bash
df -hT | grep ftp
```

```
/dev/mapper/wgroup-ftp  xfs        49G   33M   49G    1% /var/ftp
```

6）查看 test.txt 文件有没有受影响

```bash
cat test.txt
```

7）设置开机自动挂载

```bash
echo "/dev/mapper/wgroup-ftp /var/ftp xfs defaults 0 0" >> /etc/fstab
```

```
/dev/mapper/wgroup-ftp /var/ftp xfs defaults,nofail 0 0
```

> 说明：在开机中，如果找不到这个挂载点，系统就不能正常开机，为了避免，可以在 default 后面加上 `,nofail`，即使找不到这个挂载点，系统还能正常开机。

```bash
mount -a    # 让配置立即生效，不报错就设置成功
```

然后重启虚拟机验证一下：

```bash
df -hT | grep -v tmpfs
```

11. 删除逻辑卷

1）先卸载文件系统

```bash
umount /var/ftp
```

2）再删除逻辑卷

```bash
lvremove /dev/wgroup/ftp
```

```
Do you really want to remove active logical volume wgroup/ftp? [y/n]: y
  Logical volume "ftp" successfully removed
```

3）再删除卷组

```bash
vgremove wgroup
```

```
  Volume group "wgroup" successfully removed
```

4）删除物理卷

```bash
pvremove /dev/sd{b,c,d,e}
```
